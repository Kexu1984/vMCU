# AC780x TIMER Driver

这是 AC780x 系列芯片的 TIMER（定时器）驱动程序，采用三层架构设计。

## 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                        测试层 (Test Layer)                   │
│  ┌─────────────────┐  ┌─────────────────────────────────────┐│
│  │ timer_test_     │  │      multitask_timer.c              ││
│  │ wrapper.c       │  │   (多任务测试实现)                    ││
│  │ (测试包装器)      │  │                                   ││
│  └─────────────────┘  └─────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│                       驱动层 (Driver Layer)                  │
│  ┌─────────────────────────────────────────────────────────┐│
│  │              ac780x_timer.c/.h                          ││
│  │           (TIMER 业务逻辑实现)                            ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│                    硬件抽象层 (HAL Layer)                     │
│  ┌───────────────┐  ┌─────────────────────────────────────┐│
│  │ac780x_timer_  │  │      ac780x_timer_reg.h             ││
│  │hal.c/.h       │  │    (寄存器定义和映射)                  ││
│  │(硬件抽象接口)    │  │                                   ││
│  └───────────────┘  └─────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

## 文件结构

```
timer/output/c_driver/
├── Makefile                    # 构建系统
├── README_AC780x.md           # 本文档

│   ├── ac780x_timer.c        # TIMER 驱动实现

│   ├── ac780x_timer_hal.c    # HAL 实现
│   ├── ac780x_timer_hal.h    # HAL 头文件
│   └── ac780x_timer_reg.h    # TIMER 寄存器定义
└── test/                     # 测试层
    ├── timer_test_wrapper.c  # 测试包装器
    └── multitask_timer.c     # 多任务测试实现
```

## 构建系统

### 编译目标

- `all` - 构建所有组件（默认）
- `test` - 构建测试可执行文件
- `libs` - 仅构建 HAL 和驱动库文件
- `multitask` - 构建独立的多任务测试程序
- `clean` - 清理构建产物
- `help` - 显示帮助信息

### 编译选项

- `DEBUG=1` - 启用调试模式，使用调试符号和调试库
- `REAL_CHIP=1` - 为真实硬件编译（直接内存访问）
- `CC=<compiler>` - 指定编译器（默认：gcc）

### 编译示例

```bash
# 基本编译
make

# 调试版本编译
make DEBUG=1

# 为真实芯片编译
make REAL_CHIP=1

# 仅编译库文件
make libs

# 编译独立多任务测试
make multitask

# 清理构建产物
make clean

# 显示帮助
make help
```

## 运行环境

### 仿真模式（默认）

在仿真模式下，驱动通过 `libdrvintf.a` 或 `libdrvintf_dbg.a` 与 Python 设备模型通信：

```bash
# 标准模式
./ac780x_timer_test

# 调试模式
./ac780x_timer_test

# 指定基地址
./ac780x_timer_test 0x40011000
```

### 真实硬件模式

通过 `REAL_CHIP=1` 编译后，驱动直接访问硬件寄存器：

```bash
make REAL_CHIP=1
./ac780x_timer_test 0x40011000
```

## 开发指南

### 添加新的 TIMER 功能

1. **HAL 层修改**：
   - 在 `hal/ac780x_timer_hal.h` 中添加新的 HAL 接口
   - 在 `hal/ac780x_timer_hal.c` 中实现接口

2. **驱动层修改**：
   - 在 `driver/ac780x_timer.h` 中添加新的驱动 API
   - 在 `driver/ac780x_timer.c` 中实现功能

3. **测试层修改**：
   - 在 `test/multitask_timer.c` 中添加测试用例
   - 更新 `test/timer_test_wrapper.c` 中的测试流程

### 调试建议

1. **使用调试模式**：
   ```bash
   make clean && make DEBUG=1
   ```

2. **检查寄存器访问**：
   - HAL 层提供了统一的寄存器访问接口
   - 在仿真模式下可以通过日志查看寄存器操作

3. **分层调试**：
   - HAL 层：验证寄存器读写
   - 驱动层：验证业务逻辑
   - 测试层：验证整体功能

## TIMER 模块特性

### 支持的功能

- 基本定时器功能
- 多通道支持（最多4个通道）
- 中断处理
- 回调机制
- 计数器控制

### 寄存器映射

- 基地址：`0x40011000`
- 控制器通道：`TIMER_CHANNEL0` ~ `TIMER_CHANNEL3`
- 中断号：`TIMER_CHANNEL_IRQn` (18)

### API 接口

详见 `driver/ac780x_timer.h` 中的函数定义和注释。

## 集成说明

### 与 Python 设备模型集成

在仿真模式下，该驱动可以与 Python 设备模型无缝集成：

1. Python 模型负责 TIMER 行为仿真
2. C 驱动通过接口层与 Python 通信
3. 支持实时的寄存器状态同步

### 与上层应用集成

```c
#include "ac780x_timer.h"

// 初始化 TIMER
TIMER_Init();

// 配置定时器
TIMER_Config(channel, config);

// 启动定时器
TIMER_Start(channel);
```

## 故障排除

### 常见问题

1. **编译错误**：
   - 确保所有依赖文件存在
   - 检查编译器版本兼容性

2. **链接错误**：
   - 确保接口库文件存在
   - 检查库文件路径设置

3. **运行时错误**：
   - 检查基地址设置是否正确
   - 验证寄存器访问权限

驱动支持多级别日志输出，可以通过编译选项控制：

```bash
make DEBUG=1  # 启用详细日志
```
